How It Works — Step by Step
A beginner-friendly explanation of every layer, with real code snippets
1
User types a command on the frontend
The web dashboard (HTML + JavaScript) captures your text and sends it to a Python server using a simple fetch() call. Think of this as texting your instructions to the AI.
// frontend sends command to Python backend const response = await fetch('/run-command', { method: 'POST', body: JSON.stringify({ command: "open settings" }) });
2
Python backend receives it & takes a screenshot
A Flask server gets the command. Before asking the AI what to do, it first takes a screenshot of the phone using ADB — so the AI can actually "see" the current screen.
# Python (Flask) backend import subprocess def take_screenshot(): # ADB command — takes photo of phone screen subprocess.run(['adb', 'shell', 'screencap', '/sdcard/screen.png']) subprocess.run(['adb', 'pull', '/sdcard/screen.png']) return 'screen.png'
3
Claude AI looks at the screen and decides what to do
The screenshot + your command are sent to the Claude API. Claude reads both and responds with a specific action: "tap the Settings icon at coordinates (540, 320)". This is the AI brain doing its job.
# Call Claude API with screenshot + command import anthropic, base64 client = anthropic.Anthropic() with open('screen.png', 'rb') as f: img = base64.b64encode(f.read()).decode() response = client.messages.create( model="claude-sonnet-4-5-20250929", messages=[{ "role": "user", "content": [ {"type": "image", "source": {"type": "base64", "media_type": "image/png", "data": img}}, {"type": "text", "text": "User wants to: open settings. What should I tap?"} ] }] ) # Claude replies: "Tap the Settings icon at (540, 320)"
4
ADB executes the action on the real Android device
The backend sends the AI's instruction directly to the phone via ADB. This physically taps the screen, swipes, types text, or presses buttons — on the real device!
# Execute the tap on the Android device def tap(x, y): subprocess.run(['adb', 'shell', 'input', 'tap', str(x), str(y)]) def type_text(text): subprocess.run(['adb', 'shell', 'input', 'text', text]) def swipe(x1, y1, x2, y2): subprocess.run(['adb', 'shell', 'input', 'swipe', str(x1), str(y1), str(x2), str(y2)]) # Execute what Claude decided tap(540, 320) # taps the Settings icon
5
Screenshot taken again → AI checks → repeat until done
After every action, a new screenshot goes back to Claude. It checks "did that work?" and plans the next step. This loop (Think → Act → Observe) continues until the full task is complete. This is called a ReAct Agent loop.
# The agent loop — runs until task is complete while not task_complete: screenshot = take_screenshot() # 📸 observe action = ask_claude(screenshot, # 🧠 think task, history) if action['type'] == 'tap': tap(action['x'], action['y']) # 👆 act elif action['type'] == 'done': task_complete = True # ✅ finished
